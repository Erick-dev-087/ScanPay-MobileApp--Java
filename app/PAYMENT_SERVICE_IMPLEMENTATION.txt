## SUMMARY OF CHANGES MADE

### 1. Updated PaymentRequest Class
Location: `app/android-app/app/src/main/java/com/scanpay/app/data/request/PaymentRequest.java`

**What Changed:**
- Added new `qr_code_id` field (int) as primary identifier
- Added new constructor: `PaymentRequest(int qrCodeId, double amount)`
- Kept legacy constructors for backward compatibility

**Why:**
Backend expects `qr_code_id` (integer) and `amount` fields, but the app was sending `merchant_code` (string).
The fix ensures correct data is sent to the backend endpoint: POST /api/payment/initiate

**Why:**
Backend API expects `payload` field when scanning QR codes, not `qr_data`.

## KEY DIFFERENCES FROM PREVIOUS IMPLEMENTATION

### Before:
- App sent: `merchant_code` (string like "123456"), `phone`, `amount`
- Backend expected: `qr_code_id` (integer), `amount`
- Result: Backend couldn't find QR code → Connection timeout or error

### After:
- App sends: `qr_code_id` (integer from QR), `amount`
- Backend receives exactly what it expects
- Result: Successful payment initiation → Returns transaction with all details

## HOW THE FLOW WORKS NOW

1. **QR Scanning:**
   - User scans QR → QRScannerActivity parses it
   - Could call `/api/qr/scan` to get official QR ID (optional, for validation)
   - Pass QR ID to PaymentConfirmationActivity

2. **Payment Initiation:**
   - User enters amount in PaymentConfirmationActivity
   - App creates: `PaymentRequest(qrCodeId, amount)`
   - Sends POST to `/api/payment/initiate`
   - Backend validates QR, creates transaction, initiates M-Pesa push

3. **Response Handling:**
   - Backend returns: `message`, `transaction`, `checkout_request_id`
   - App updates TransactionResponse model
   - Navigates to PaymentSuccessActivity on success

## DATABASE REQUIREMENTS FOR TESTING

The backend database should have:
- At least 1 QR code record (ID: 1)
- At least 1 vendor/merchant associated with the QR
- Your logged-in user ID in the session

If tests fail with "QR not found", create a QR code first via backend:
```python
# Quick backend test
from app import db
from models import QRCode, Vendor

vendor = Vendor.query.filter_by(user_id=1).first()
if vendor:
    qr = QRCode(vendor_id=vendor.id, qr_type='STATIC', is_active=True)
    db.session.add(qr)
    db.session.commit()
    print(f"Created QR with ID: {qr.id}")
```

## TROUBLESHOOTING

### Issue: "Invalid QR Code ID" toast message
**Solution:**
- Make sure merchant_code can be parsed as integer
- Or pass qr_code_id directly through intent

### Issue: Network error in response
**Solution:**
- Check backend URL in Constants.java
- Verify backend is running
- Check firewall settings
- For emulator, use 10.0.2.2:5000

### Issue: 400 Bad Request response
**Solution:**
- Verify QR ID exists in database: `SELECT * FROM qr_codes WHERE id = 1;`
- Verify amount > 0 and <= 500000
- Check backend logs for detailed error

### Issue: 401 Unauthorized
**Solution:**
- Login again to refresh JWT token
- Token may have expired
- Check SessionManager is storing token correctly
