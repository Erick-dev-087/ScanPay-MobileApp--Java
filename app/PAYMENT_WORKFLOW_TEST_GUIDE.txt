## What Gets Tested

### Payment Request Structure
The tests verify that payment requests are sent with this structure:

```json
{
  "qr_code_id": 3,
  "amount": 500
}
```

**Fields:**
- `qr_code_id` (integer): The ID of the QR code to pay
- `amount` (double): Payment amount in KES (1 - 500,000)

### Expected Success Response (201 Created)
```json
{
  "message": "Payment initiated successfully",
  "transaction_id": 42,
  "checkout_request_id": "ws_CO_123456789",
  "amount": 500,
  "vendor": {
    "name": "Coffee Shop",
    "business_shortcode": "123456"
  },
  "transaction_type": "CustomerPayBillOnline",
  "status": "Pending",
  "instructions": "Please check your phone and enter your M-Pesa PIN to complete the payment"
}
```

### Error Responses
- `400 Bad Request`: Missing QR ID, invalid amount, QR/vendor inactive
- `401 Unauthorized`: Invalid or expired JWT token
- `404 Not Found`: QR code or vendor not found
- `500 Server Error`: Backend issue

### Common Issues & Fixes

#### Issue: Connection Timeout
**Cause:** Backend not running or unreachable
**Fix:**
1. Verify backend is running on Render
2. Check internet connectivity
3. Verify BASE_URL in Constants.java is correct

#### Issue: 401 Unauthorized
**Cause:** Invalid JWT token
**Fix:**
1. Ensure test user exists (dan@njoroge.com)
2. Check password is correct
3. Verify token hasn't expired
4. Re-login to get fresh token

#### Issue: 404 Not Found
**Cause:** QR code or vendor doesn't exist
**Fix:**
1. Verify TEST_QR_CODE_ID exists in database
2. Run seed data script if needed
3. Check that vendor is active

#### Issue: 400 Bad Request
**Cause:** Invalid payment parameters
**Fix:**
1. Check amount is between 1-500,000 KES
2. Verify QR Code ID is valid integer
3. Ensure QR code is active (not disabled)

## Integration with Mobile App

### How the App Uses Payment API

1. **User scans QR code** → QR code ID extracted
2. **User enters amount** → Validation (1-500,000 KES)
3. **User clicks Pay** → `PaymentConfirmationActivity.initiatePayment()` called
4. **App creates PaymentRequest** with qr_code_id and amount
5. **ApiService makes POST request** to `/api/payment/initiate`
6. **JWT token added** by AuthInterceptor automatically
7. **Response received** → Success/Error handling
8. **Success** → Navigate to `PaymentSuccessActivity`
9. **Error** → Show error toast and retry

### Key Files Involved

- `PaymentRequest.java` - Request model
- `PaymentConfirmationActivity.java` - UI for payment
- `ApiService.java` - Retrofit interface
- `ApiClient.java` - Retrofit configuration
- `Constants.java` - BASE_URL configuration
- `TransactionResponse.java` - Response model

## API Endpoint Reference

### POST /api/payment/initiate

**URL:** `https://qr-pay-system.onrender.com/api/payment/initiate`

**Authentication:** Required (JWT Bearer Token)

**Request Body:**
```json
{
  "qr_code_id": 3,
  "amount": 500
}
```

**Success Response (201 Created):**
```json
{
  "message": "Payment initiated successfully",
  "transaction_id": 42,
  "checkout_request_id": "ws_CO_123456789",
  "amount": 500,
  "vendor": {
    "name": "Coffee Shop",
    "business_shortcode": "123456"
  },
  "transaction_type": "CustomerPayBillOnline",
  "status": "Pending",
  "instructions": "Please check your phone and enter your M-Pesa PIN to complete the payment"
}
```

**Error Responses:**
- `400 Bad Request` - Invalid input
- `401 Unauthorized` - Missing/invalid token
- `404 Not Found` - Resource not found
- `500 Server Error` - Backend error
